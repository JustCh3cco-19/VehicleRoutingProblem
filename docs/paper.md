# Paper workflow

The academic paper is maintained as LaTeX under `docs/paper/`.

## Directory layout

```text
docs/paper/main.tex                         Document preamble and section inclusion
docs/paper/sections/introduction.tex        Problem and ACO overview
docs/paper/sections/openmp_mpi.tex          Parallel CPU implementation
docs/paper/sections/cuda.tex                CUDA implementation
docs/paper/sections/performance_comparison.tex  Experimental method and conclusions
docs/paper/scripts/analyze_results.py       Validation, aggregation, and plotting
docs/paper/generated/analysis.md            Generated validation narrative
docs/paper/generated/summaries/*.csv        Generated tables used to check results
docs/paper/figures/scaling/*.pdf            Strong/weak scaling figures
docs/paper/figures/comparison/*.pdf         Backend size-sweep figures
docs/paper/Makefile                         Plot and PDF targets
report.pdf                             Final PDF written at repository root
```

## Regenerate experimental material

From the repository root:

```bash
make -C docs/paper plots
```

The command runs `docs/paper/scripts/analyze_results.py --root .`. The script reads
the current files under `results/manual_campaign/`, validates solution routes,
writes summaries below `docs/paper/generated/`, and replaces the seven expected PDF
figures below `docs/paper/figures/`. It does not modify raw result files.

The checked-in campaign CSVs currently reference `instances/test_aligned/`,
which is absent from the repository. Consequently, plot regeneration stops
with `mandatory strong OpenMP baseline is missing` after route validation
excludes those rows. Restore the exact referenced instances or produce a
consistent result set before running `plots`; do not substitute instances only
because their filenames match.

The paper includes the figure PDFs by relative path from
`docs/paper/sections/performance_comparison.tex`. Generated CSV summaries support the
reported statistics but are not included as LaTeX tables.

## Compile the PDF

The checked-in figures already satisfy the paper dependencies. Build with:

```bash
make -C docs/paper
```

The paper Makefile runs `pdflatex` twice in a temporary directory and copies
the result to `report.pdf` at the repository root. `make pdfs` from the root
builds the same paper PDF.

The LaTeX installation must provide every package imported by `main.tex`,
including `microtype`, `setspace`, `titlesec`, the algorithm packages,
`placeins`, `balance`, and `tikz-cd`.

Clean paper build products with:

```bash
make -C docs/paper clean
```

The checked-in LaTeX sources are not generated by the result pipeline.
